\section{Introduzione}

Boopie Share è una piattaforma software per la condivisione e gestione di contenuti musicali quali spartiti, documenti, file audio, video e collegamenti a risorse esterne.

Il sistema consente agli utenti di:
\begin{itemize}
    \item Pubblicare contenuti multimediali
    \item Associare metadati (autori, generi...)
    \item Inserire commenti ed annotazioni
    \item Interagire con altri utenti tramite follow
\end{itemize}

L’applicazione è strutturata secondo un’architettura client–server multilivello implementata tramite framework Flask lato server.
(questa sezione verrà approfondità meglio più avanti)

\section{Requisiti}

\subsection{Prerequisiti}

Il sistema è suddiviso in componente Client e Server.

La comunicazione tra Client e Server avviene tramite richieste HTTP verso endpoint RESTful esposti dal Server Flask.

All’avvio del Client:
\begin{itemize}
    \item viene verificata la versione del software;
    \item vengono controllate eventuali dipendenze Python mancanti;
    \item vengono eventualmente aggiornate versioni o dipendenze
    \item viene stabilita la connessione con il server.
\end{itemize}

Il sistema è compatibile con Windows, macOS e Linux.

\subsection{Gerarchia dei Ruoli}

Il sistema implementa una gerarchia di livelli utente rappresentata tramite enumerazione \textit{UserLevel}.  
I livelli sono ordinati gerarchicamente dal più privilegiato al meno privilegiato:

\begin{itemize}
    \item \textbf{Root (0)}: controllo totale del sistema.
    \item \textbf{Admin (1)}: gestione utenti e contenuti.
    \item \textbf{Moderatore (2)}: moderazione dei contenuti e applicazione restrizioni.
    \item \textbf{Publisher (3)}: pubblicazione e gestione dei propri contenuti.
    \item \textbf{Regular (4)}: utente standard con permessi limitati.
    \item \textbf{Restricted (5)}: utente con accesso ridotto (sola visualizzazione).
    \item \textbf{Banned (6)}: utente impossibilitato ad accedere al sistema.
\end{itemize}

Il livello numerico inferiore indica un livello di privilegio maggiore.


\subsection{Requisiti Non Funzionali}

\begin{itemize}
    \item Sicurezza: password memorizzate tramite hashing.
    \item Scalabilità: separazione in layer indipendenti.
    \item Manutenibilità: architettura modulare.
    \item Portabilità: indipendenza dal sistema operativo.
    \item Performance: ricerca indicizzata sui media.
\end{itemize}

\section{Architettura del Sistema}

\subsection{Architettura Generale}

Il sistema adotta un’architettura client–server multilivello:

\begin{itemize}
    \item Client Web
    \item Server Flask
    \item Database relazionale
\end{itemize}

\subsection{Architettura Flask}

Il flusso di una richiesta è il seguente:

\begin{enumerate}
    \item Il client invia una richiesta HTTP.
    \item Flask intercetta la richiesta tramite routing.
    \item Il Dispatcher (Front Controller) identifica il comando.
    \item Il Service Layer esegue la logica applicativa.
    \item Il Data Access Layer interagisce con il database.
    \item La risposta JSON viene restituita al client.
\end{enumerate}
